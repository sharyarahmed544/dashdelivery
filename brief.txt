01. 
PROJECT OVERVIEW
Dash Delivery is an established UK & European courier company requiring a world-class digital platform that converts visitors into customers, enables self-serve bookings, generates professional PDF documentation, and gives staff full control over every aspect of the website and business operations through a comprehensive admin panel.
This brief covers the complete technical specification for Antigravity Studio. The existing HTML prototype (dash-delivery-v3.html) has been provided as the visual design reference and must be faithfully converted into a production-grade, full-stack web application.
1.1 Project Goals
Convert the approved HTML/CSS prototype into a pixel-perfect, production Next.js application
Build a comprehensive Node.js + PostgreSQL backend with full REST API
Implement a fully featured Admin Panel (CMS) where every piece of content is editable
Deliver PDF Invoice & Quote generation with Dash branding
Implement Pickup Booking, Quote Request, and Contact systems
Integrate email notification system for all transactional events
Achieve 95+ Lighthouse scores across Performance, SEO, and Accessibility
Deploy on scalable cloud infrastructure with CI/CD pipeline

05. 
ADMIN PANEL — FULL SPECIFICATION
The admin panel is accessible at /admin — protected by JWT authentication. It must be a separate React SPA (within the same Next.js project) with its own layout, sidebar navigation, and role-based access controls. All data shown in the admin must reflect the live database in real time.
IMPORTANT: Every section of the public website that contains text, images, pricing, or content must have a corresponding editable entry in the admin panel. There must be zero hard-coded content on the frontend that cannot be changed without a developer.
5.1 Admin Login & Authentication
Login page at /admin/login — email + password, JWT issued on success
Refresh token rotation — 15-minute access token, 7-day refresh token stored in httpOnly cookie
Role-based access: SUPER_ADMIN (all access), ADMIN (most access), STAFF (view + limited edit)
"Forgot Password" flow — sends reset link via email, 1-hour expiry token
Login audit log — records IP, device, timestamp for every successful and failed login
Forced logout after 8 hours of inactivity
2FA (Two-Factor Authentication) via TOTP authenticator app — optional but available

SECURITY REQUIREMENTS
Security Concern
Implementation Required
SQL Injection
Prisma ORM with parameterised queries — no raw SQL with user input
XSS Prevention
All user input sanitised with DOMPurify server-side; CSP headers set
CSRF Protection
SameSite=Strict cookies; CSRF tokens on state-changing forms
Rate Limiting
Express-rate-limit on all API routes; Redis-backed for distributed deploy
Authentication
JWT with short expiry (15m); httpOnly refresh token; no tokens in localStorage
Password Security
Bcrypt with minimum cost factor 12; minimum 8 chars + complexity requirements
File Upload Security
File type whitelist (images only for uploads); size limit 5MB; virus scan optional
API Key Protection
All third-party API keys in environment variables only — never in client bundle
HTTPS
Enforced at Cloudflare level; HSTS header; HTTP→HTTPS redirect
GDPR
Cookie consent banner; privacy policy page; data deletion endpoint for users
PCI Compliance
No card data ever touches Dash servers — Stripe handles all card processing
Security Headers
Helmet.js on Express: X-Frame-Options, X-Content-Type-Options, HSTS, CSP
Dependency Audits
npm audit in CI pipeline — fail build on high-severity vulnerabilities
Logging
All admin actions logged to audit_log table; no passwords or tokens ever logged
